# Production Dependencies for AMA Cryptography 3.1+
# Install with: pip install -r requirements.txt
#
# INVARIANT-1: Core cryptographic operations have ZERO external Python
# crypto dependencies.  The native C library (libama_cryptography) provides
# every primitive used by the runtime path:
#   - Ed25519 digital signatures (RFC 8032)
#   - AES-256-GCM authenticated encryption (NIST SP 800-38D)
#   - HKDF-SHA3-256 key derivation (RFC 5869) — INVARIANT-7: no Python fallback
#   - SHA3-256 / HMAC-SHA3-256 / SHA-512 hashing
#   - Constant-time comparison utilities (dudect-verified)
#   - ML-DSA-65 (FIPS 204), ML-KEM-1024 (FIPS 203), SLH-DSA (FIPS 205)
# Build with: cmake -B build -DAMA_USE_NATIVE_PQC=ON && cmake --build build
#
# This file is intentionally minimal because pyproject.toml's [project]
# table is the single source of truth for declared dependencies.  Listing
# them twice creates drift; this file exists only as a discoverability
# breadcrumb for downstream tools that grep for requirements.txt.

# Secure memory operations (OPTIONAL, recommended for high-security environments)
# The secure_memory module uses the AMA native C backend or POSIX mlock/munlock.
# Install with: pip install "ama-cryptography[secure-memory]"
# pynacl>=1.5.0

# HSM integration (OPTIONAL)
# PKCS#11 support for hardware security modules.
# Install with: pip install "ama-cryptography[hsm]"
# PyKCS11>=1.5.18

# Mathematical monitoring dependencies (OPTIONAL)
# Core cryptographic operations work without numpy/scipy.
# Install with: pip install "ama-cryptography[monitoring]"
# numpy>=1.24.0,<3.0.0
# scipy>=1.7.0
